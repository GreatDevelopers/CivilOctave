43 <p>This page explains how to interpret the graphs that are generated by doxygen.</p>
44 <p>Consider the following example: </p><div class="fragment"><div class="line"><a name="l00001"></a><span class="lineno"> 1</span>&#160;/*! Invisible class because of truncation */</div>
45 <div class="line"><a name="l00002"></a><span class="lineno"> 2</span>&#160;class Invisible { };</div>
46 <div class="line"><a name="l00003"></a><span class="lineno"> 3</span>&#160;</div>
47 <div class="line"><a name="l00004"></a><span class="lineno"> 4</span>&#160;/*! Truncated class, inheritance relation is hidden */</div>
48 <div class="line"><a name="l00005"></a><span class="lineno"> 5</span>&#160;class Truncated : public Invisible { };</div>
49 <div class="line"><a name="l00006"></a><span class="lineno"> 6</span>&#160;</div>
50 <div class="line"><a name="l00007"></a><span class="lineno"> 7</span>&#160;/* Class not documented with doxygen comments */</div>
51 <div class="line"><a name="l00008"></a><span class="lineno"> 8</span>&#160;class Undocumented { };</div>
52 <div class="line"><a name="l00009"></a><span class="lineno"> 9</span>&#160;</div>
53 <div class="line"><a name="l00010"></a><span class="lineno"> 10</span>&#160;/*! Class that is inherited using public inheritance */</div>
54 <div class="line"><a name="l00011"></a><span class="lineno"> 11</span>&#160;class PublicBase : public Truncated { };</div>
55 <div class="line"><a name="l00012"></a><span class="lineno"> 12</span>&#160;</div>
56 <div class="line"><a name="l00013"></a><span class="lineno"> 13</span>&#160;/*! A template class */</div>
57 <div class="line"><a name="l00014"></a><span class="lineno"> 14</span>&#160;template&lt;class T&gt; class Templ { };</div>
58 <div class="line"><a name="l00015"></a><span class="lineno"> 15</span>&#160;</div>
59 <div class="line"><a name="l00016"></a><span class="lineno"> 16</span>&#160;/*! Class that is inherited using protected inheritance */</div>
60 <div class="line"><a name="l00017"></a><span class="lineno"> 17</span>&#160;class ProtectedBase { };</div>
61 <div class="line"><a name="l00018"></a><span class="lineno"> 18</span>&#160;</div>
62 <div class="line"><a name="l00019"></a><span class="lineno"> 19</span>&#160;/*! Class that is inherited using private inheritance */</div>
63 <div class="line"><a name="l00020"></a><span class="lineno"> 20</span>&#160;class PrivateBase { };</div>
64 <div class="line"><a name="l00021"></a><span class="lineno"> 21</span>&#160;</div>
65 <div class="line"><a name="l00022"></a><span class="lineno"> 22</span>&#160;/*! Class that is used by the Inherited class */</div>
66 <div class="line"><a name="l00023"></a><span class="lineno"> 23</span>&#160;class Used { };</div>
67 <div class="line"><a name="l00024"></a><span class="lineno"> 24</span>&#160;</div>
68 <div class="line"><a name="l00025"></a><span class="lineno"> 25</span>&#160;/*! Super class that inherits a number of other classes */</div>
69 <div class="line"><a name="l00026"></a><span class="lineno"> 26</span>&#160;class Inherited : public PublicBase,</div>
70 <div class="line"><a name="l00027"></a><span class="lineno"> 27</span>&#160; protected ProtectedBase,</div>
71 <div class="line"><a name="l00028"></a><span class="lineno"> 28</span>&#160; private PrivateBase,</div>
72 <div class="line"><a name="l00029"></a><span class="lineno"> 29</span>&#160; public Undocumented,</div>
73 <div class="line"><a name="l00030"></a><span class="lineno"> 30</span>&#160; public Templ&lt;int&gt;</div>
74 <div class="line"><a name="l00031"></a><span class="lineno"> 31</span>&#160;{</div>
75 <div class="line"><a name="l00032"></a><span class="lineno"> 32</span>&#160; private:</div>
76 <div class="line"><a name="l00033"></a><span class="lineno"> 33</span>&#160; Used *m_usedClass;</div>
77 <div class="line"><a name="l00034"></a><span class="lineno"> 34</span>&#160;};</div>
78 </div><!-- fragment --><p> This will result in the following graph:</p>
79 <center><div class="image">
80 <img src="graph_legend.png" />
81 </div>
82 </center><p>The boxes in the above graph have the following meaning: </p>
83 <ul>
84 <li>
85 A filled gray box represents the struct or class for which the graph is generated. </li>
86 <li>
87 A box with a black border denotes a documented struct or class. </li>
88 <li>
89 A box with a gray border denotes an undocumented struct or class. </li>
90 <li>
91 A box with a red border denotes a documented struct or class forwhich not all inheritance/containment relations are shown. A graph is truncated if it does not fit within the specified boundaries. </li>
92 </ul>
93 <p>The arrows have the following meaning: </p>
94 <ul>
95 <li>
96 A dark blue arrow is used to visualize a public inheritance relation between two classes. </li>
97 <li>
98 A dark green arrow is used for protected inheritance. </li>
99 <li>
100 A dark red arrow is used for private inheritance. </li>
101 <li>
102 A purple dashed arrow is used if a class is contained or used by another class. The arrow is labeled with the variable(s) through which the pointed class or struct is accessible. </li>
103 <li>
104 A yellow dashed arrow denotes a relation between a template instance and the template class it was instantiated from. The arrow is labeled with the template parameters of the instance. </li>
105 </ul>
